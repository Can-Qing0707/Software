## 🟠 高（严重影响）

### 13. Login 将原始数据库错误泄漏给客户端
- **文件**: `backend/internal/service/auth_service.go:26-32`
- **描述**: `FindByUsername` 返回非 `ErrRecordNotFound` 错误时，原始 GORM 错误直接返回给客户端，暴露数据库连接信息
- **修复**: 真实错误记日志，返回 `"系统繁忙，请稍后再试"`

### 14. 注册时 FindByUsername 的错误被丢弃
- **文件**: `backend/internal/service/auth_service.go:61`
- **描述**: `existing, _ := s.userRepo.FindByUsername(req.Username)` 丢弃错误，真实 DB 故障被掩盖
- **修复**: 检查错误，仅 `gorm.ErrRecordNotFound` 视为"不存在"

### 15. 登出不撤销 JWT Token
- **文件**: `backend/internal/handler/auth_handler.go:46-48`
- **描述**: Logout 仅返回 200，无 Token 黑名单/撤销机制，被盗 Token 在 72 小时内仍可认证
- **修复**: 实现 Token 黑名单（内存/Redis），中间件检查黑名单

### 16. JWT 时间戳使用本地时间而非 UTC
- **文件**: `backend/pkg/jwt/jwt.go:23-24`
- **描述**: `time.Now()` 返回服务器本地时间，时区变化或夏令时会导致签发/过期时间不一致
- **修复**: 改为 `time.Now().UTC()`

### 17. 上传文件无服务端类型校验
- **文件**: `backend/internal/handler/upload_handler.go:19-51`
- **描述**: 后端未校验 Content-Type 或扩展名，攻击者可上传 `.exe`、`.sh` 等任意类型文件
- **修复**: 校验文件扩展名和 Content-Type 白名单

### 18. 前端上传设置错误的 Content-Type
- **文件**: `frontend/src/api/submission.js:13`
- **描述**: 手动设置 `Content-Type: multipart/form-data` 缺少 boundary 参数，浏览器应自行设置
- **修复**: 移除显式 Content-Type 头

### 19. 前端报表生成请求缺少必填字段
- **文件**: `frontend/src/views/report/ReportView.vue:107-115`
- **描述**: 后端 `GenerateReportReq` 需要 `type` 和 `format` 字段（binding: required），前端只发送 `course_id` 和 `task_id`，后端会返回 400
- **修复**: 前端添加报告类型和格式选择 UI，补全请求字段

### 20. 提交状态机 evaluated 状态从未设置
- **文件**: `backend/internal/repository/submission_repo.go:59-70`, `backend/internal/service/evaluation_service.go`
- **描述**: `parsed` 状态已通过 #5 修复（核查时文档解析后设置）；但 `evaluated` 状态在教师/LLM 评分完成后从未被设置
- **修复**: 教师评分和 LLM 评分完成后更新提交状态为 `evaluated`

- **文件**: `backend/internal/service/verify_service.go`

- **修复**: 使用数据库事务，状态更新和结果创建原子化

### 27. LLM 服务无 HTTP 状态码检查
- **文件**: `backend/internal/service/llm_service.go:65-89`

- **修复**: 先检查状态码，非 200 时读取响应体返回明确错误

### 28. LLM 配置无运行时校验
- **文件**: `backend/internal/service/llm_service.go:46`
- **描述**: `Chat()` 不校验 `APIURL`、`APIKey`、`Model` 是否为空，空值导致不可预期的 HTTP 请求
- **修复**: 调用前检查必填字段，空时返回明确错误

### 29. Config handler 将 DB 错误当作"未找到"处理
- **文件**: `backend/internal/handler/config_handler.go:23-27`
- **描述**: 任意 DB 错误都返回 `Success(c, nil)`（HTTP 200），掩盖真实故障
- **修复**: 区分 `gorm.ErrRecordNotFound` 和真实 DB 错误

### 30. GetUserID 上下文缺失时返回 0 无错误
- **文件**: `backend/internal/middleware/auth.go:56-62`

- **修复**: 改为 401

### 32. DashboardStats DTO 定义了但从未使用

- **描述**: 结构体定义存在，但无 handler/service 使用它，路由中也无 dashboard API
- **修复**: 要么实现 dashboard stats API，要么删除死代码

### 33. 路由不将已验证用户从 /login 重定向走
- **文件**: `frontend/src/router/index.js:108-117`
